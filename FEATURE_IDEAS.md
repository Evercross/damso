# FEATURE_IDEAS.md — Damso 기능 확장 기획

> damso의 현재 기반(음성→텍스트 삽입)을 활용해서 추가할 수 있는 기능들을
> 난이도와 사용자 임팩트 기준으로 정리한 문서입니다.

---

## 현재 damso가 할 수 있는 것

음성 녹음 → STT 변환 → 텍스트 정제(구두점/용어 치환) → 활성 앱에 삽입

이 파이프라인의 각 단계에 기능을 끼워넣거나, 파이프라인 자체를 확장하는 방향.

---

## Tier 1: 빠르게 추가 가능 (1-2일, 기존 코드 수정 수준)

### 1-1. 음성 입력 피드백 강화
**지금:** 메뉴바 아이콘 색상 변경으로만 상태 표시
**추가:** 변환된 텍스트를 macOS 알림(notification)으로 미리보기
- 삽입 전에 "이렇게 입력됩니다" 확인 가능
- 오인식 시 취소 기회 제공
- 난이도: 낮음 (rumps.notification() 한 줄)
- 임팩트: ★★★ — 오입력 방지, 신뢰감 향상

### 1-2. 최근 입력 히스토리 빠른 접근
**지금:** history.py에 SQLite로 기록은 하지만 접근이 불편
**추가:** 메뉴바 드롭다운에 최근 5개 입력 표시 → 클릭하면 재삽입
- 같은 문구 반복 입력할 때 유용 (회의 중 반복 메모 등)
- 난이도: 낮음 (rumps 메뉴 아이템 추가)
- 임팩트: ★★☆ — 편의성 향상

### 1-3. 음성 입력 자동 언어 감지
**지금:** 설정에서 언어 수동 선택
**추가:** Qwen3-ASR이 자동 감지 지원하므로 "Auto" 모드 추가
- 한국어/영어 섞어 쓸 때 편리
- 난이도: 낮음 (stt.py 옵션 추가)
- 임팩트: ★★★ — 언어 전환 번거로움 제거

### 1-4. 용어 사전 학습 모드
**지금:** 프리셋 76개 + 수동 추가
**추가:** 자주 수정하는 패턴을 자동 감지 → "이 치환 규칙 추가할까요?" 제안
- 예: "클로드"가 매번 "클라우드"로 인식되면 자동 제안
- 난이도: 중간 (패턴 감지 로직 필요)
- 임팩트: ★★★ — 쓸수록 정확도 향상

---

## Tier 2: 의미있는 확장 (3-5일, 새 모듈 추가)

### 2-1. 음성 메모 모드 (텍스트 파일 저장)
**지금:** 활성 앱에 텍스트 삽입만 가능
**추가:** "메모 모드" — 음성을 텍스트로 변환해서 날짜별 마크다운 파일로 저장
- `~/.damso/memos/2026-04-06.md`에 타임스탬프와 함께 누적
- 아이디어 캡처, 회의록, 일기 등
- Obsidian/Notion 연동 가능성
- 난이도: 중간 (새 저장 경로 + 모드 전환 UI)
- 임팩트: ★★★ — 용도 확장, 독립 앱 가치 상승

### 2-2. 텍스트 후처리 파이프라인
**지금:** 구두점 + 용어 치환만
**추가:** 변환 후 텍스트에 추가 처리 옵션
- 존댓말 ↔ 반말 변환 (Claude API 연동)
- 영어 → 한국어 번역 후 삽입
- 요약 모드 (긴 음성을 핵심만 정리)
- 포맷 모드 (마크다운, 이메일 형식 등)
- 난이도: 중간~높음 (LLM API 연동 필요)
- 임팩트: ★★★ — "단순 STT"에서 "AI 음성 비서"로 진화

### 2-3. 앱별 프로필
**지금:** 모든 앱에 동일한 설정 적용
**추가:** 앱별로 다른 설정 자동 적용
- 카카오톡: 반말 + 빠른 삽입
- 메일: 존댓말 + 구두점 꼼꼼히
- 메모: 마크다운 포맷
- 난이도: 중간 (config 구조 확장 + 활성 앱 감지는 이미 있음)
- 임팩트: ★★☆ — 파워유저 기능

### 2-4. 음성 명령 (Voice Command)
**지금:** 음성 = 텍스트 입력만
**추가:** 특정 키워드로 앱 제어
- "담소 취소" → 마지막 입력 되돌리기 (Cmd+Z)
- "담소 줄바꿈" → Enter 키 삽입
- "담소 복사" → 현재 선택 텍스트 복사
- 난이도: 중간 (키워드 감지 + CGEvent 액션 매핑)
- 임팩트: ★★☆ — 핸즈프리 워크플로우

## Tier 3: 킬러 기능 (1-2주, 아키텍처 확장)

### 3-1. 실시간 자막 모드 (Live Caption)
**지금:** 녹음 완료 후 일괄 변환
**추가:** 실시간 스트리밍 변환 — 말하는 동안 텍스트가 실시간으로 나타남
- 플로팅 오버레이 윈도우에 실시간 자막 표시
- 회의/강의 실시간 메모
- 난이도: 높음 (스트리밍 ASR + 오버레이 UI)
- 임팩트: ★★★ — 완전히 새로운 사용 시나리오

### 3-2. 시스템 오디오 캡처 (회의록)
**지금:** 마이크 입력만
**추가:** macOS 시스템 오디오 캡처 → 줌/팀즈 회의 자동 기록
- BlackHole(가상 오디오 디바이스) 연동
- 화자 분리 (Speaker Diarization)
- 난이도: 높음 (가상 오디오 + 화자 분리 모델)
- 임팩트: ★★★ — 수익화 가능한 킬러 기능

### 3-3. Claude API 연동 — 음성 AI 비서
**지금:** 음성 → 텍스트
**추가:** 음성 → Claude API → 응답을 텍스트로 삽입
- "이 이메일에 답장 써줘" → 음성으로 맥락 설명 → Claude가 답장 생성 → 삽입
- 난이도: 중간 (API 호출 자체는 쉬움, UX 설계가 핵심)
- 임팩트: ★★★ — damso만의 차별점

### 3-4. Shortcuts/Automator 연동
**지금:** 독립 앱
**추가:** macOS Shortcuts에서 damso 기능 호출 가능
- "Hey Siri, 담소로 메모해" → 음성 녹음 → 텍스트 저장
- Shortcuts 액션으로 다른 자동화와 연결
- 난이도: 높음 (Shortcuts 통합은 .app 패키징 필요)
- 임팩트: ★★☆ — macOS 생태계 통합

## 추천 우선순위 (사용 패턴 기준)

### 즉시 (Phase 1 안정화와 함께)
1. **1-1. 알림 미리보기** — 오입력 방지, 가장 빠르게 체감
2. **1-3. 자동 언어 감지** — 한/영 섞어 쓸 때 편리
3. **1-2. 최근 히스토리** — 반복 입력 시 시간 절약

### 안정화 후 (Phase 2)
4. **2-1. 음성 메모 모드** — 아이디어 캡처, 블로그 소재 수집에 활용
5. **2-2. 텍스트 후처리** — Claude API 연동으로 가치 상승
6. **1-4. 사전 학습 모드** — 장기적으로 정확도 향상

### 킬러 기능 (Phase 3 — 배포 준비 시)
7. **3-1. 실시간 자막** — 사용 시나리오 확장
8. **3-3. Claude AI 비서** — damso만의 차별점
9. **3-2. 회의록** — 수익화 가능성

## 수익화 관점 메모

damso가 유료 앱이 되려면 "무료 STT 앱과 뭐가 다른가"를 답해야 함.
현재 macOS 내장 받아쓰기(Dictation)와의 차별점:

| | macOS 내장 | damso |
|--|-----------|-------|
| 오프라인 | ✅ | ✅ (MLX) |
| 한국어 | ✅ | ✅ |
| 용어 치환 | ❌ | ✅ |
| 히스토리 | ❌ | ✅ |
| 후처리(AI) | ❌ | 🔜 |
| 앱별 프로필 | ❌ | 🔜 |
| 실시간 자막 | ❌ | 🔜 |
| 회의록 | ❌ | 🔜 |

**핵심 차별점은 "AI 후처리"와 "회의록"** — 이 두 가지가 있어야 유료 가치가 생김.
